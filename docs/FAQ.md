## 一般的な質問

### Q1: Junmai AutoDev とは何ですか？

**A**: Junmai AutoDev は、写真家のワークフロー全体を自動化する AI 搭載の Lightroom 現像支援システムです。撮影後の写真を指定フォルダにコピーするだけで、取り込み・選別・現像・書き出しまでを自動的に実行します。

### Q2: 無料で使えますか？

**A**: はい、完全無料です。OSS LLM（Ollama + Llama 3.1）を使用しており、OpenAI API キーなどの有料サービスは一切不要です。

### Q3: どのくらいの時間を節約できますか？

**A**: 100 枚の写真を処理する場合、従来は約 105 分かかっていた作業が、実作業時間 5 分程度に短縮されます（約 95% の時間削減）。

### Q4: プロの写真家でなくても使えますか？

**A**: はい、趣味で写真を撮る方から、プロの写真家まで、幅広く使用できます。初心者向けのプリセットも用意されています。

### Q5: 既存の Lightroom ワークフローに影響しますか？

**A**: いいえ、既存のワークフローを妨げません。仮想コピーとスナップショットを使用するため、元の写真は常に保護されます。

## システム要件

### Q6: 必要な GPU は何ですか？

**A**: NVIDIA RTX 4060 8GB VRAM 以上を推奨します。RTX 3060 12GB や RTX 4070 以上でも動作します。

### Q7: AMD GPU や Intel GPU でも動作しますか？

**A**: 現在は NVIDIA GPU（CUDA 対応）のみサポートしています。将来的に AMD GPU（ROCm）のサポートを検討しています。

### Q8: GPU がない PC でも使えますか？

**A**: CPU のみでも動作しますが、処理速度が大幅に低下します（1 枚あたり 30 秒以上）。GPU の使用を強く推奨します。

### Q9: macOS でも動作しますか？

**A**: はい、macOS 12 以上で動作します。ただし、Apple Silicon（M1/M2/M3）では GPU 加速が制限される場合があります。

### Q10: どのくらいのストレージが必要ですか？

**A**: 最低 50GB の空き容量を推奨します。RAW ファイルのサイズや処理枚数によって必要容量は増加します。

## インストールとセットアップ

### Q11: インストールは難しいですか？

**A**: 基本的な PC 操作ができれば、インストールガイドに従って 30 分程度でセットアップできます。

### Q12: Lightroom のどのバージョンが必要ですか？

**A**: Lightroom Classic 12.0 以上が必要です。Lightroom CC（クラウド版）には対応していません。

### Q13: Python の知識は必要ですか？

**A**: いいえ、Python の知識は不要です。インストールガイドに従ってコマンドをコピー＆ペーストするだけです。

### Q14: Redis とは何ですか？なぜ必要ですか？

**A**: Redis はバックグラウンド処理のジョブキューとして使用されます。複数の写真を効率的に並行処理するために必要です。

### Q15: Ollama とは何ですか？

**A**: Ollama は、ローカルで LLM（大規模言語モデル）を実行するためのツールです。完全無料で、インターネット接続なしで動作します。

## 機能について

### Q16: どのような写真に対応していますか？

**A**: RAW ファイル（CR3, CR2, NEF, ARW, RAF, ORF, DNG）と JPEG に対応しています。

### Q17: 動画ファイルも処理できますか？

**A**: いいえ、現在は静止画のみ対応しています。

### Q18: 複数のカメラで撮影した写真を混在させても大丈夫ですか？

**A**: はい、異なるカメラで撮影した写真も自動的に判別して適切に処理します。

### Q19: 連写した写真はどうなりますか？

**A**: 類似写真グループとして自動的にグループ化され、最良の 1 枚が選択されます。

### Q20: 手動で現像設定を調整できますか？

**A**: はい、承認キューで「修正」を選択すると、Lightroom で手動調整できます。調整内容は学習データとして記録されます。

### Q21: プリセットをカスタマイズできますか？

**A**: はい、独自のプリセットを追加したり、既存のプリセットを調整したりできます。

### Q22: 複数の書き出しフォーマットに対応していますか？

**A**: はい、SNS 用、印刷用、アーカイブ用など、複数の書き出しプリセットを同時に設定できます。

### Q23: クラウドストレージへの自動アップロードは可能ですか？

**A**: はい、Dropbox、Google Drive、OneDrive、FTP/SFTP に対応しています。

### Q24: モバイルから操作できますか？

**A**: はい、スマートフォンやタブレットから進捗確認と承認操作が可能です。

### Q25: 複数の PC で同時に使用できますか？

**A**: いいえ、Lightroom カタログは同時に 1 つの PC からのみアクセスできます。

## AI と LLM

### Q26: AI はどのように写真を評価しますか？

**A**: ピント精度、露出適正、構図バランス、被写体評価を総合的に分析し、1-5 星で評価します。

### Q27: AI の評価は正確ですか？

**A**: 初期状態で約 80% の精度があり、使用するほど学習して精度が向上します。

### Q28: AI の評価基準を変更できますか？

**A**: はい、`config.json` で選別閾値を調整できます。また、学習機能により、あなたの好みに自動的に適応します。

### Q29: どの LLM モデルを使用していますか？

**A**: デフォルトでは Llama 3.1 8B Instruct を使用しています。11B や Mixtral 8x7B にも変更可能です。

### Q30: LLM モデルを変更するとどうなりますか？

**A**: より大きなモデル（11B, Mixtral）は評価精度が向上しますが、処理速度が低下し、より多くの VRAM が必要になります。

### Q31: インターネット接続は必要ですか？

**A**: 初回のモデルダウンロード時のみ必要です。その後は完全オフラインで動作します。

### Q32: データはどこに送信されますか？

**A**: データは一切外部に送信されません。すべての処理がローカルで完結します。

### Q33: 顔認識機能はありますか？

**A**: はい、OpenCV による顔検出機能があります。ポートレート写真の評価に使用されます。

### Q34: 顔データは保存されますか？

**A**: 顔の位置情報のみが一時的に使用され、顔画像自体は保存されません。

### Q35: AI の学習データはどこに保存されますか？

**A**: ローカルの SQLite データベース（`local_bridge/data/processing.db`）に保存されます。

## Lightroom 連携

### Q36: Lightroom Classic 以外のバージョンでも使えますか？

**A**: いいえ、現在は Lightroom Classic のみ対応しています。Lightroom CC（クラウド版）には対応していません。

### Q37: 既存の Lightroom カタログを使用できますか？

**A**: はい、既存のカタログをそのまま使用できます。

### Q38: 複数のカタログを切り替えて使用できますか？

**A**: はい、`config.json` でカタログパスを変更することで、複数のカタログに対応できます。

### Q39: 仮想コピーとは何ですか？

**A**: Lightroom の機能で、元の写真を複製せずに異なる現像設定を適用できます。ディスク容量を節約できます。

### Q40: スナップショットとは何ですか？

**A**: 現像設定の履歴を保存する機能です。失敗時に元の状態に戻すことができます。

### Q41: 元の写真が上書きされることはありますか？

**A**: いいえ、RAW ファイル自体は決して変更されません。現像設定のみが Lightroom カタログに保存されます。

### Q42: Lightroom のコレクションは自動作成されますか？

**A**: はい、セッションごとにコレクションが自動作成されます。

### Q43: Lightroom のキーワードは自動付与されますか？

**A**: はい、EXIF データとコンテキストに基づいてキーワードが自動付与されます。

### Q44: Lightroom プラグインは安全ですか？

**A**: はい、Lua SDK の公式 API のみを使用しており、システムファイルにはアクセスしません。

### Q45: Lightroom が起動していなくても処理できますか？

**A**: バックグラウンド処理（取り込み、AI 評価）は Lightroom 非起動時も実行されますが、現像設定の適用には Lightroom の起動が必要です。

## パフォーマンス

### Q46: 1 枚の写真の処理にどのくらい時間がかかりますか？

**A**: RTX 4060 で平均 2-5 秒です。GPU やモデルによって変動します。

### Q47: 同時に何枚まで処理できますか？

**A**: デフォルトでは 3 枚まで並行処理します。`config.json` で変更可能です。

### Q48: 処理中に他の作業はできますか？

**A**: はい、バックグラウンドで処理されるため、他の作業を妨げません。CPU 使用率も自動調整されます。

### Q49: GPU 温度が高くなりすぎることはありますか？

**A**: GPU 温度が設定値（デフォルト 75°C）を超えると、自動的に処理が一時停止します。

### Q50: メモリ不足になることはありますか？

**A**: 16GB RAM があれば通常は問題ありません。メモリ不足時は自動的にキャッシュがクリアされます。

### Q51: 処理速度を向上させる方法はありますか？

**A**: 以下の方法があります:
- より高性能な GPU を使用
- 軽量な LLM モデル（8B）を使用
- 量子化を有効化
- SSD を使用

### Q52: 大量の写真（1000 枚以上）を一度に処理できますか？

**A**: はい、可能です。ただし、処理時間は枚数に比例します（1000 枚で約 1-2 時間）。

### Q53: 処理を一時停止できますか？

**A**: はい、デスクトップ GUI から一時停止・再開が可能です。

### Q54: 処理の優先度を変更できますか？

**A**: はい、AI 評価スコアが高い写真や、手動投入したジョブが優先的に処理されます。

### Q55: キャッシュはどのくらいの容量を使用しますか？

**A**: Redis キャッシュは最大 2GB まで使用します。定期的に自動クリアされます。

## セキュリティとプライバシー

### Q56: 写真データは外部に送信されますか？

**A**: いいえ、すべての処理がローカルで完結します。クラウド同期を有効にしない限り、外部への送信は一切ありません。

### Q57: EXIF データの GPS 情報は保護されますか？

**A**: はい、GPS 情報は処理にのみ使用され、外部には送信されません。匿名化オプションもあります。

### Q58: 顔認識データはどう扱われますか？

**A**: 顔の位置情報のみが一時的に使用され、顔画像自体は保存されません。

### Q59: API キーは安全に保存されますか？

**A**: はい、API キー（クラウド同期用など）は環境変数または暗号化された設定ファイルに保存されます。

### Q60: ログファイルに機密情報は含まれますか？

**A**: いいえ、ログにはファイルパスと処理結果のみが記録され、写真の内容や個人情報は含まれません。

### Q61: 複数ユーザーで使用する場合、データは分離されますか？

**A**: 現在は単一ユーザー向けです。複数ユーザーで使用する場合は、別々の PC または仮想環境を推奨します。

### Q62: データベースは暗号化されていますか？

**A**: デフォルトでは暗号化されていませんが、オプションで暗号化を有効にできます。

### Q63: バックアップは自動的に作成されますか？

**A**: はい、データベースは日次で自動バックアップされます。

### Q64: 商用利用時のライセンスは？

**A**: MIT ライセンスに従い、商用利用も自由です。

### Q65: GDPR に準拠していますか？

**A**: ローカル処理のため、GDPR の対象外です。ただし、クラウド同期を使用する場合は、各クラウドサービスの規約を確認してください。

## トラブルシューティング

### Q66: インストール中にエラーが発生しました

**A**: [トラブルシューティングガイド](TROUBLESHOOTING.md#インストール関連) を参照してください。

### Q67: Redis に接続できません

**A**: Redis が起動しているか確認してください:
```bash
redis-cli ping
```

### Q68: Ollama に接続できません

**A**: Ollama が起動しているか確認してください:
```bash
ollama serve
```

### Q69: GPU が認識されません

**A**: CUDA が正しくインストールされているか確認してください:
```bash
nvidia-smi
```

### Q70: 処理が途中で止まります

**A**: ログファイル（`local_bridge/logs/errors.log`）を確認してください。GPU メモリ不足やディスク容量不足の可能性があります。

### Q71: 写真が取り込まれません

**A**: ホットフォルダーのパスが正しいか、書き込み権限があるか確認してください。

### Q72: AI 評価が実行されません

**A**: Celery ワーカーが起動しているか確認してください:
```bash
python start_worker.py
```

### Q73: Lightroom プラグインが動作しません

**A**: プラグインマネージャーで有効になっているか、バックエンドサーバーが起動しているか確認してください。

### Q74: モバイル Web UI にアクセスできません

**A**: ファイアウォールでポート 5100 が許可されているか、同じネットワークに接続されているか確認してください。

### Q75: エラーメッセージの意味がわかりません

**A**: エラーメッセージを [トラブルシューティングガイド](TROUBLESHOOTING.md) で検索するか、GitHub Issues で質問してください。
